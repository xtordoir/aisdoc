About Service API

The entities managed by the Services API are:

  • Facility: The organization exposing services to users
  • Service: The services exposed in the system
  • Order: The orders submitted by the users as customers of a Facility

The typical usage flow is the following:

  1. A user with rights to manage Services within a Facility creates a Service, defining all parameters a customer has to fill-in for a Order to be processed. Usually this task is performed in the web application.

  2. A user with rights to manage Services within a Facility creates a Service, defining all parameters a customer has to fill-in for a Order to be processed. Usually this task is performed in the web application.

Services are offered by Facilities and exposed in the web application to their customers. Services are configured by a service manager. A Service allows to register the required information from the customer for an Order to be executed.

For example, a computing service might register an input file and a number of parameters. For web-lab Services, samples and their parameters can be registered and associated with an experiment. The Services API is used by the Facility executing the Orders to get the data into its own IT system and to update the status of the Orders.
